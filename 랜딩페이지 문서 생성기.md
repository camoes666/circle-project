### 랜딩페이지 문서 생성기

랜딩페이지 생성 범용 프롬프트 프레임워크

> **사용법**: 이 프롬프트를 AI에 그대로 붙여넣고, [ ] 안의 항목을 채우거나 AI가 질문하도록 설정하세요.

① 의도 수집 레이어 (Input Layer)

아래 항목에 먼저 답하거나, AI에게 순서대로 질문하게 하세요.

```
당신은 전환율 최적화(CRO) 전문 랜딩페이지 카피라이터입니다.
아래 9가지 질문에 대한 답을 바탕으로, 랜딩페이지 전체 구성과 문구를 설계합니다.

[Q1] 제품/서비스명과 한 줄 설명은?
→ 예: "SlideEdit - AI가 만든 HTML 프레젠테이션을 즉시 수정하는 편집기"

[Q2] 핵심 타겟 고객은 누구이며, 그들의 가장 큰 Pain Point는?
→ 예: "AI로 HTML 슬라이드를 만들지만 수정할 도구가 없어 매번 코드를 직접 건드리는 실무자"

[Q3] 이 랜딩페이지의 단 하나의 1차 전환 목표(Primary CTA)는?
→ 반드시 페이지에서 바로 측정 가능한 행동으로 작성
→ 예: "무료 체험 시작", "데모 보기", "베타 대기 등록", "상담 예약"

[Q3-1] 이 페이지의 사업 목표(Business Goal)는?
→ 1차 CTA와 구분하여 작성
→ 예: "초기 사용자 확보", "체험 후 유료 전환", "특정 워크플로 락인", "세일즈 콜 연결"

[Q4] 핵심 가치 제안(Value Proposition) 3가지는?
→ 예: "① AI 기반 슬라이드 수정 ② 빠른 이미지 편집 ③ 별도 설치 없이 바로 사용"

[Q5] 신뢰를 높일 수 있는 증거 자산은 무엇인가요?
→ 강한 사회적 증거가 없더라도 아래 중 있는 것을 모두 작성:
  - 사용자 수 / 대기자 수 / 재방문율
  - 창업자·팀 전문성
  - 데모 영상 / 스크린샷 / 전후 비교
  - 베타 사용자 코멘트
  - 언론 / 파트너 / 커뮤니티 언급
  - 보증 / 환불 / 무료 정책

[Q6] 가격/오퍼 구조는 어떻게 되나요?
→ 예: "베타 무료", "14일 무료 체험", "월 19달러", "상담 후 견적"
→ 가격이 없더라도 '왜 지금 결제가 필요 없는지'를 설명할 수 있으면 작성

[Q7] 고객이 지금 대신 쓰고 있는 대안은 무엇인가요?
→ 예: "VS Code", "PowerPoint", "Figma", "도구 없이 직접 HTML 수정"

[Q8] 제품을 보는 즉시 이해시키는 핵심 장면 1개는 무엇인가요?
→ 예: "AI가 만든 HTML 슬라이드를 클릭 한 번으로 텍스트·이미지를 바로 수정하는 화면"

[Q9] 유입 환경과 제약은 무엇인가요?
→ 예: "모바일 광고 유입", "커뮤니티 링크 클릭", "처음 듣는 제품", "체류 시간 짧음"
```

② 구조 생성 레이어 (Structure Layer)

위 입력을 바탕으로 AI에게 아래 명령을 실행시킵니다.

```
위 Q1~Q9 답변을 기반으로 아래 8개 섹션을 순서대로 설계하세요.
각 섹션마다 [섹션명 / 목적 / 핵심 카피 공식 / 완성 문구 / 데스크톱 배치 제안 / 모바일 배치 제안] 형태로 출력하세요.

--- SECTION 1: HERO (히로 섹션) ---
목적: 5초 내 핵심 가치 전달 + 이탈 방지
카피 공식: [타겟의 욕망] + [구체적 수치/결과] + [행동 동사 CTA]
필수 요소:
  - H1 헤드라인: 타겟이 원하는 결과를 직접 서술 (15자 이내 권장)
  - 서브헤드라인: 헤드라인을 1-2문장으로 구체화
  - 시각 증거 1개: 제품 화면·GIF·데모 영상 중 하나 (없으면 전후 비교 텍스트로 대체)
  - CTA 버튼 문구: 동사+이익 구조 ("무료로 시작하기", "지금 30% 할인받기")
  - CTA 일치 규칙: 버튼 문구와 클릭 후 첫 화면의 약속이 정확히 일치해야 함
    예: "무료 체험 시작"이면 다음 화면은 즉시 체험 또는 간단 가입이어야 함
  - 신뢰 마이크로카피: 버튼 하단 마찰 제거 문구 ("신용카드 불필요 · 설치 없음 · 3분 내 시작")
  - 보조 신뢰 바: 증거 자산 중 가장 강한 것 1개 배치 (숫자, 전문성, 언론 등)
검증 체크:
  □ 헤드라인만 읽어도 '누구를 위한 무엇인지' 파악 가능한가? (Y/N)
  □ CTA 문구와 실제 다음 단계가 일치하는가? (Y/N)
  □ 시각 자료 없이도 이해되지만, 있으면 이해 속도가 빨라지는가? (Y/N)

--- SECTION 2: PROBLEM AGITATION (문제 공감) ---
목적: 타겟의 Pain Point를 언어화하여 공감 형성
카피 공식: PAS (Problem → Agitate → Solution 암시)
필수 요소:
  - 공감형 소제목: "혹시 이런 경험 있으신가요?"
  - Pain Point 3-4개 불릿 (타겟의 언어로, 감정적으로)
  - 기존 대안(Q7)을 직접 또는 간접적으로 언급
    예: "VS Code를 열고 태그를 찾고 있나요?", "매번 PPT로 다시 만들고 계신가요?"
  - 전환 브릿지 문장: "이제 다른 방법이 있습니다"
검증 체크:
  □ 타겟이 읽으며 "맞아, 나 얘기다"라고 느낄 수 있는가? (Y/N)
  □ 기존 대안 대비 고통이 구체적으로 서술되었는가? (Y/N)

--- SECTION 3: SOLUTION (솔루션 제시) ---
목적: 제품/서비스가 문제를 어떻게 해결하는지 명확히 제시
카피 공식: BAB (Before → After → Bridge)
필수 요소:
  - 솔루션 헤드라인: "[제품명]은 [문제]를 [방법]으로 해결합니다"
  - Before/After 대비 표현
  - 작동 방식 3단계 (Step 1 → 2 → 3, 각 10자 이내)
  - 기존 대안(Q7)과의 차별점 1문장
검증 체크:
  □ 솔루션이 Pain Point와 1:1로 대응되는가? (Y/N)
  □ 기존 대안 대신 이 제품을 써야 하는 이유가 명확한가? (Y/N)

--- SECTION 4: SOCIAL PROOF (사회적 증거) ---
목적: 전환 불안 해소 + 신뢰 구축
카피 공식: STAR (Situation → Task → Action → Result)
필수 요소:
  - 섹션 헤드라인: 수치 포함 ("1,200명이 이미 선택했습니다")
  - 후기/사례 카드 3개: 직함+이름+구체적 결과 포함
  - 로고 배너 (언론/파트너사, 있을 경우)
  - 핵심 지표 3개: 숫자로 표현

[사회적 증거가 약한 경우 대체 규칙]
후기가 부족하다면 아래 조합으로 대체 가능:
  ① 베타 사용자 한 줄 피드백 2개
  ② 창업자/팀의 전문성 또는 배경 1개
  ③ 제품 전후 비교 사례 1개
  ④ 초기 지표 (대기자 수, 데모 신청 수, 총 사용 시간 등)
  ⑤ 지원 응답 속도, 공개 로드맵, 환불 정책
→ "왜 지금 믿어도 되는지"를 설명하는 신뢰 캡션 1문장을 반드시 포함

검증 체크:
  □ 후기 숫자가 약해도 신뢰 자산이 비어 있지 않은가? (Y/N)
  □ 후기에 구체적 수치나 결과가 포함되어 있는가? (Y/N)

--- SECTION 5: FEATURES/BENEFITS (기능과 혜택) ---
목적: 이성적 판단 근거 제공
카피 공식: Feature → Benefit → Proof
필수 요소:
  - 섹션 헤드라인: 혜택 중심 ("당신이 얻게 될 것들")
  - 기능 카드 4-6개: 아이콘 + 기능명(짧게) + 혜택 설명(1문장)
  - 각 기능은 Pain Point와 연결되어야 함
검증 체크:
  □ '기능'이 아닌 '혜택(나에게 어떤 좋은 점)'으로 서술되었는가? (Y/N)

--- SECTION 5.5: OFFER / PRICING (가격과 오퍼) ---
※ SaaS / 앱 / 구독형 제품이면 필수, 아니면 조건부 포함
목적: 가격 불안 제거 + CTA 직전 망설임 해소
카피 공식: Offer → Risk Reversal → Next Step
필수 요소:
  - 가격 또는 무료 정책 명시
  - 무료 체험/베타 범위와 제한 설명
  - "어떤 사람이 어떤 플랜을 선택하면 되는지" 1문장
  - CTA 버튼 1개
  - 환불/해지/카드 필요 여부 마이크로카피
  - 가격이 없는 경우: "왜 지금 결제가 필요 없는지" 명확히 서술
검증 체크:
  □ 가격/무료 정책이 CTA 이전에 충분히 설명되었는가? (Y/N)
  □ CTA 직전의 비용 불안을 줄였는가? (Y/N)

--- SECTION 6: FAQ (자주 묻는 질문) ---
목적: 구매 전 마지막 장벽 제거
카피 공식: 반론 선제 처리 (Objection Handling)
필수 요소:
  - Q&A 5-7개 (가격, 난이도, 환불, 비교, 지원 관련)
  - 질문은 타겟이 실제 쓸 언어로 작성
  - 답변은 간결하고 직접적으로 (2-3문장)
  - 아래 3가지 질문은 반드시 포함:
    ① "지금 쓰는 [대안 도구] 대신 왜 이걸 써야 하나요?"
    ② "무료로 어디까지 가능한가요?"
    ③ "제 기존 작업물과 어떻게 연결되나요?"
검증 체크:
  □ 가장 흔한 구매 거부 이유 3가지가 모두 다루어졌는가? (Y/N)
  □ 경쟁 대안 대비 질문이 포함되어 있는가? (Y/N)

--- SECTION 7: FINAL CTA (최종 행동 촉구) ---
목적: 결정을 미루는 방문자에게 마지막 전환 유도
카피 공식: 긴급성 + 손실 회피 + 재확인
필수 요소:
  - 강화된 헤드라인: Hero보다 더 구체적인 이익을 덧붙인 표현
  - 핵심 가치 제안 3줄 재요약 (체크마크 형태)
    → 3줄 중 1줄은 반드시 비용/시간 손실 회피 문장으로 작성
  - CTA 버튼 (Hero와 동일 약속 유지, 표현은 더 강하게)
  - 긴급성/희소성 마이크로카피 (있을 경우)
  - 최종 신뢰 요소: 보증, 환불, 무료 정책
검증 체크:
  □ Final CTA가 Hero보다 더 구체적이거나 동등하게 강한가? (Y/N)
  □ 이 섹션만 봐도 전환할 이유가 충분한가? (Y/N)
```

③ 검증 레이어 (Validation Layer)

섹션 생성 후 아래 체크리스트를 AI에게 자가 검증하게 합니다.

```
생성된 랜딩페이지 콘텐츠를 아래 28개 항목으로 검증하고,
각 항목을 ✅(통과) / ⚠️(수정 필요) / ❌(미충족)으로 표시한 뒤
⚠️와 ❌ 항목에 대해 즉시 개선안을 제시하세요.

[명확성]
□ 헤드라인만으로 타겟과 가치가 5초 내 전달되는가
□ 전문 용어 없이 타겟이 이해할 수 있는 언어인가
□ 모든 섹션에 명확한 목적이 있는가

[설득력]
□ Pain Point가 타겟의 실제 언어로 서술되었는가
□ 기존 대안 대비 왜 이 제품을 써야 하는지가 명확한가
□ 솔루션이 Pain Point와 1:1 대응되는가
□ 기능이 '혜택' 언어로 번역되었는가
□ 구체적 수치가 최소 3개 이상 포함되었는가

[신뢰성]
□ 사회적 증거에 구체적 결과가 포함되었는가
□ 증거가 약하면 대체 신뢰 자산으로 공백 없이 채워졌는가
□ 신뢰 요소가 Above-the-fold 근처에 최소 1개 배치되었는가
□ 반론(FAQ)이 선제적으로 처리되었는가
□ 보증/환불 정책이 명시되었는가

[전환 최적화]
□ CTA가 페이지 내 최소 3회 이상 등장하는가
□ CTA 버튼 문구가 동사+이익 구조인가
□ CTA 문구와 클릭 후 다음 화면의 약속이 일치하는가
□ 버튼 하단에 마찰 제거 마이크로카피가 있는가
□ 가격/무료 범위가 CTA 직전에 충분히 설명되었는가
□ 선택지가 단일 CTA로 집중되어 있는가 (분산 없음)

[구조/흐름]
□ 섹션 순서가 AIDA 또는 PAS 흐름을 따르는가
□ 각 섹션 말미에 다음 섹션으로 이어지는 브릿지가 있는가
□ Above-the-fold에 핵심 가치+CTA가 모두 포함되었는가

[모바일/마찰]
□ 모바일에서 헤드라인, 핵심 가치, CTA가 1~2스크린 안에 들어오는가
□ 입력 폼이 최소화되어 있는가 (또는 폼 길이의 이유가 정당한가)
□ 버튼/카피/레이아웃이 모바일 한 손 사용 기준에 무리 없는가
□ 시각 자료가 이해 속도를 높이도록 배치되었는가

[완성도]
□ 누락된 섹션이 없는가 (8섹션 체크)
□ 타겟 페르소나가 전체 카피에 일관되게 반영되었는가
□ 최종 CTA가 Hero CTA보다 강하거나 동등한가
```

④ 출력 형식 지정 (Output Spec)

```
검증 완료 후, 아래 형식으로 최종 콘텐츠를 출력하세요.

출력 형식:
- 각 섹션을 명확히 구분 (--- SECTION N: [섹션명] ---)
- 모든 카피는 실제 사용 가능한 완성 문장으로 작성
- 대괄호 플레이스홀더([제품명] 등)는 실제 내용으로 대체
- 각 섹션 하단에:
```
