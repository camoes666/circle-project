# SubToDoc 제품 기획서

## WHO — 누구를 위한 제품인가

**페르소나: 개발자 지훈 (가상)**

- **상황:** 사이드 프로젝트 아이디어를 찾거나 기술을 공부하며 영어 유튜브를 자주 보는 개발자
- **목표:** 영상 내용을 빠르게 문서로 정리해 나중에 참고하고 싶다
- **고통:** 긴 영상을 다 보기 부담스럽고, 내용을 따로 정리하는 게 귀찮다
- **현재 해결 방식:** 직접 노트 정리하거나 그냥 북마크만 해두고 안 본다

---

## WHAT — 어떤 문제를 푸는가

**Problem Statement**
> 영어 영상을 소비하는 사용자는 내용을 정리하고 싶지만, 직접 노트를 쓰는 마찰이 너무 커서 결국 북마크만 쌓인다. 근본 원인은 "나중에 정리하자"는 미루기가 구조적으로 발생하기 때문이다.

**JTBD**
> 개발자는 영어 유튜브 영상을 볼 때, 핵심 내용을 한국어 문서로 빠르게 뽑아내기 위해 이 제품을 고용한다.

**경쟁 서비스(릴리스 AI 등)가 해결하지 못하는 이유**
- 서버 기반 서비스는 API 비용이 발생하거나 구독이 필요함
- 본인의 API 키를 직접 써서 비용을 직접 통제하고 싶은 파워유저 니즈를 충족하지 못함
- 서버리스 구조로 인해 데이터가 외부로 나가지 않는다는 프라이버시 이점 없음

## HOW — 어떻게 해결하는가

**선택된 방향: 심플 변환기 (A안)**
URL 입력 → 형식 선택 → 문서 생성. 빠른 개발과 낮은 진입 장벽에 집중.

**기술 스택**

| 항목 | 선택 |
|------|------|
| 프레임워크 | React + Vite |
| 스타일링 | Tailwind CSS |
| AI API | 사용자 직접 입력 (OpenAI / Claude 등) |
| 자막 추출 | YouTube 자막 API (클라이언트 사이드) |
| 저장 | localStorage |
| 배포 | GitHub Pages |

**기능 목록 (MoSCoW) — v1 완료 기준 업데이트**

| 분류 | 기능 | 상태 |
|------|------|------|
| **Must** | YouTube URL 입력 → 자막 추출 → AI 문서 생성 | ✅ 완료 |

### v2 기능 상세

#### 1. 영상 미리보기 임베드
- URL 입력란에 유효한 YouTube URL 감지 즉시 `youtube.com/embed/{videoId}` iframe 표시
- 회의록·노트 형식에서 특히 유용 — 영상 보면서 변환 결과 동시 확인
- 모바일에서는 접기/펼치기 토글

#### 2. 타임스탬프 포함 모드
- Supadata API가 반환하는 `offset`(초) 값을 `[MM:SS]` 형식으로 자막에 삽입
- AI가 중요 내용마다 가까운 타임스탬프를 결과에 포함
- 결과 뷰어에서 `[MM:SS]` 클릭 → `youtube.com/watch?v=ID&t=초` 새 탭 이동
- 자막 가져오기 방법이 로컬 서버·자동 모드일 때는 XML의 `start` 속성 활용

#### 3. 직접 지시하기
- 변환 버튼 위에 작은 토글 ("직접 지시하기 ▾")
- 열면 텍스트 입력란 표시, 예시 힌트: "초보자 친화적으로", "유머러스하게", "스타트업 투자자용으로"
- AI 프롬프트 끝에 `추가 지시: {입력값}` 삽입

#### 4. 변환 히스토리
- 변환 성공 시 localStorage에 자동 저장 (최대 20개, LIFO)
- 저장 항목: URL, videoId, 형식, 변환 결과, 타임스탬프, 커스텀 지시
- 메인 화면 하단 "지난 변환" 아코디언 패널
- 항목 클릭 → URL·형식·결과 복원 / 삭제 버튼 제공

**MVP 정의 (v1)**
API 키 입력 → YouTube URL 입력 → 문서 형식 선택 → 문서 생성. ✅ 달성

**v2 목표**
자막 보면서 영상도 확인 + 원하는 톤으로 지시 + 이전 작업 바로 재활용.

**플랫폼**
웹 앱 (데스크탑 브라우저 우선)
