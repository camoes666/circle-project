# SubToDoc 제품 기획서

## WHO — 누구를 위한 제품인가

**페르소나: 개발자 지훈 (가상)**

- **상황:** 사이드 프로젝트 아이디어를 찾거나 기술을 공부하며 영어 유튜브를 자주 보는 개발자
- **목표:** 영상 내용을 빠르게 문서로 정리해 나중에 참고하고 싶다
- **고통:** 긴 영상을 다 보기 부담스럽고, 내용을 따로 정리하는 게 귀찮다
- **현재 해결 방식:** 직접 노트 정리하거나 그냥 북마크만 해두고 안 본다

---

## WHAT — 어떤 문제를 푸는가

**Problem Statement**
> 영어 영상을 소비하는 사용자는 내용을 정리하고 싶지만, 직접 노트를 쓰는 마찰이 너무 커서 결국 북마크만 쌓인다. 근본 원인은 "나중에 정리하자"는 미루기가 구조적으로 발생하기 때문이다.

**JTBD**
> 개발자는 영어 유튜브 영상을 볼 때, 핵심 내용을 한국어 문서로 빠르게 뽑아내기 위해 이 제품을 고용한다.

**경쟁 서비스(릴리스 AI 등)가 해결하지 못하는 이유**
- 서버 기반 서비스는 API 비용이 발생하거나 구독이 필요함
- 본인의 API 키를 직접 써서 비용을 직접 통제하고 싶은 파워유저 니즈를 충족하지 못함
- 서버리스 구조로 인해 데이터가 외부로 나가지 않는다는 프라이버시 이점 없음

---

## HOW — 어떻게 해결하는가

**선택된 방향: 심플 변환기 (A안)**
URL 입력 → 형식 선택 → 문서 생성. 빠른 개발과 낮은 진입 장벽에 집중.

**기술 스택**

| 항목 | 선택 |
|------|------|
| 프레임워크 | React 18 + Vite |
| 스타일링 | Tailwind CSS |
| AI API | 사용자 직접 입력 (Groq / OpenAI / Gemini) |
| 자막 추출 | Supadata API / 로컬 Python 서버 / 자동(CORS Worker) |
| 저장 | localStorage |
| 배포 | GitHub Pages |

## v2 기능 상세 (✅ 완료)

### 1. 영상 미리보기 임베드
- URL 입력 즉시 `youtube.com/embed/{videoId}` iframe 표시
- `useMemo`로 url → videoId 실시간 파생

### 2. 타임스탬프 포함 모드
- Supadata `offset` 값을 `[MM:SS]`로 삽입 → AI 결과에 포함
- ResultViewer에서 `[MM:SS]` 클릭 → `youtube.com/?t=초` 이동

### 3. 직접 지시하기
- 토글 클릭 → 입력란 펼침 → `buildPrompt` options에 추가 지시 삽입

### 4. 변환 히스토리
- 변환 성공 시 localStorage 자동 저장 (최대 20개)
- 썸네일·날짜·형식 표시, 클릭 시 전체 복원

## v3 기능 상세 (🔜 예정)

### 1. 마인드맵
- 중심 주제 → 대분류 → 소분류 계층 구조로 정리
- 결과 뷰어: 들여쓰기 트리 UI (CSS 기반, 외부 라이브러리 없음)
- 사용 시나리오: 강의·개념 영상 → XMind/Miro에 붙여넣기용

### 2. 슬라이드 (PPT용)
- 5-8장 슬라이드 구성, 각 슬라이드 제목 + 3-5개 불릿
- 결과 뷰어: 슬라이드 카드 UI (슬라이드 번호, 제목, 내용)
- 사용 시나리오: 발표 준비용 초안, 파워포인트/키노트에 붙여넣기

### 3. 트위터 스레드
- 5-8개 트윗, 각 280자 이내
- 첫 트윗은 강렬한 훅, 마지막은 요약/CTA
- 결과 뷰어: 트윗 카드 UI (트윗 번호, 글자 수 게이지)
- 사용 시나리오: 인사이트 공유, 지식 기반 콘텐츠 제작

### 4. LinkedIn 포스트
- 훅 → 핵심 내용(이모지+불릿) → 질문/통찰 → 해시태그 구조
- 결과 뷰어: 해시태그 강조 렌더링
- 사용 시나리오: 전문 지식 공유, 네트워킹 콘텐츠
