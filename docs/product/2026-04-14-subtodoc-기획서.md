# SubToDoc 제품 기획서

## WHO — 누구를 위한 제품인가

**페르소나: 개발자 지훈 (가상)**

- **상황:** 사이드 프로젝트 아이디어를 찾거나 기술을 공부하며 영어 유튜브를 자주 보는 개발자
- **목표:** 영상 내용을 빠르게 문서로 정리해 나중에 참고하고 싶다
- **고통:** 긴 영상을 다 보기 부담스럽고, 내용을 따로 정리하는 게 귀찮다
- **현재 해결 방식:** 직접 노트 정리하거나 그냥 북마크만 해두고 안 본다

---

## WHAT — 어떤 문제를 푸는가

**Problem Statement**
> 영어 영상을 소비하는 사용자는 내용을 정리하고 싶지만, 직접 노트를 쓰는 마찰이 너무 커서 결국 북마크만 쌓인다. 근본 원인은 "나중에 정리하자"는 미루기가 구조적으로 발생하기 때문이다.

**JTBD**
> 개발자는 영어 유튜브 영상을 볼 때, 핵심 내용을 한국어 문서로 빠르게 뽑아내기 위해 이 제품을 고용한다.

**경쟁 서비스(릴리스 AI 등)가 해결하지 못하는 이유**
- 서버 기반 서비스는 API 비용이 발생하거나 구독이 필요함
- 본인의 API 키를 직접 써서 비용을 직접 통제하고 싶은 파워유저 니즈를 충족하지 못함
- 서버리스 구조로 인해 데이터가 외부로 나가지 않는다는 프라이버시 이점 없음

---

## HOW — 어떻게 해결하는가

**선택된 방향: 심플 변환기 (A안)**
URL 입력 → 형식 선택 → 문서 생성. 빠른 개발과 낮은 진입 장벽에 집중.

**기술 스택**

| 항목 | 선택 |
|------|------|
| 프레임워크 | React + Vite |
| 스타일링 | Tailwind CSS |
| AI API | 사용자 직접 입력 (OpenAI / Claude 등) |
| 자막 추출 | YouTube 자막 API (클라이언트 사이드) |
| 저장 | localStorage |
| 배포 | GitHub Pages |

**MVP 기능 목록 (MoSCoW)**

| 분류 | 기능 |
|------|------|
| **Must** | YouTube URL 입력 → 자막 추출 → AI 문서 생성 |
| **Must** | 문서 형식 선택 (요약 / 블로그 / 회의록 / 노트) |
| **Must** | API 키 입력 및 localStorage 저장 |
| **Should** | 내보내기 (Markdown, PDF) |
| **Should** | 다국어 지원 (영어 영상 → 한국어 문서) |
| **Could** | 히스토리 (localStorage 기반) |
| **Could** | 커스텀 프롬프트 |
| **Won't** | 서버, DB, 회원가입, 소셜 공유 |

**MVP 정의**
API 키 입력 → YouTube URL 입력 → 문서 형식 선택 → 문서 생성. 이것만.

**플랫폼**
웹 앱 (데스크탑 브라우저 우선)

- 회원가입 / 로그인
