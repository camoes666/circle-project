# SubToDoc Lean Canvas

| 칸 | 내용 |
|----|------|
| **문제** | 1. 영어 유튜브 영상 내용을 정리하는 마찰이 너무 크다 2. 기존 서비스는 구독/비용이 필요하다 3. 북마크만 쌓이고 실제로 재활용되지 않는다 |
| **고객 세그먼트** | 영어 유튜브를 자주 보며 자신의 API 키를 직접 쓸 수 있는 개발자 / 파워유저 |
| **고유 가치 제안** | "API 키 하나면 충분 — 서버 없이, 구독 없이, 유튜브 영상을 바로 문서로" |
| **솔루션** | URL 입력 → 자막 추출 → 형식 선택(요약/블로그/회의록/노트) → 문서 생성 → 내보내기 |
| **채널** | GitHub 오픈소스 공개 → 개발자 커뮤니티(Reddit r/programming, 개발 오픈채팅, X/Twitter 공유) |
| **수익 모델** | 없음 (사이드 프로젝트 / 오픈소스). 추후 Pro 기능 유료화 가능 |
| **비용 구조** | GitHub Pages 무료, 도메인 비용(선택), AI API 비용은 사용자 부담 |
| **핵심 지표** | GitHub Star 100개 (3개월 내), 주간 활성 사용자 50명 |
| **경쟁 우위** | 완전 클라이언트 사이드 (데이터 외부 전송 없음) + 오픈소스 + 무료 |
